# VCC Clipper — 內網跨子網 P2P 多人協作工具

## 快速開始

1. **安裝依賴：**
   ```bash
   pip install websockets
   ```

2. **啟動信令伺服器：**
   ```bash
   python3 signal_server.py
   ```

3. **開啟用戶端：**
   在多個瀏覽器分頁/視窗（最多 50 台）中開啟 `VCC_Clipper.html`。

4. **建立配對：**
   - 第一台瀏覽器點擊「建立配對」按鈕，產生一組配對代碼（四位數字）。
   - 其他瀏覽器輸入該代碼並點擊「加入配對」。
   - 所有瀏覽器自動互相建立 P2P WebRTC 網狀連線。

5. **開始使用！**
   - 連線成功後，所有成員即可使用聊天室與檔案傳輸。
   - 狀態列顯示已連線數量與您的顯示名稱。
   - 點擊頂部的顯示名稱可自訂暱稱（預設為隨機組合，如「快樂熊貓42」）。

## 功能

- 💬 **聊天室** — 即時訊息廣播，輸入文字後按 Enter 發送給所有成員。自訂顯示名稱，系統自動廣播成員加入/離開通知。
- 📁 **檔案傳輸** — 拖放上傳，支援大檔案區塊傳輸、進度條顯示、指定對象傳送與重試功能。
- 👥 **多人網狀網路** — Full Mesh 架構，支援最多 50 台同時連線，新成員自動與所有人配對。
- 🏷️ **顯示名稱系統** — 每位成員自動獲得隨機中文暱稱（形容詞+動物+數字），可點擊編輯，名稱透過 WebRTC DataChannel 交換。
- 🎯 **指定對象傳檔** — 檔案傳輸可選「所有對象」或特定一台電腦。
- 🔒 **無伺服器資料傳輸** — 僅透過信令伺服器進行初始配對，實際資料透過 WebRTC P2P 直連。

## 如何擴充新功能

新增一個 Tab：

1. 在 `<nav>` 加入新按鈕：`<button class="tab-btn" data-target="newtab">🆕 新功能</button>`
2. 在 `<main>` 加入新的 `<section data-tab="newtab">` 內容區塊
3. 在 JavaScript 的 DataChannel `onmessage` 路由中加入對應的訊息類型處理
4. 需要發送資料時，透過 `broadcastToPeers({type: 'newtab', ...})` 廣播給所有人，或 `sendToPeer(peerId, {type: ...})` 傳給指定對象。

## 手動 E2E 測試流程

### 前置準備

1. 啟動信令伺服器：`python3 signal_server.py`
2. 確認終端機顯示 `listening on ws://localhost:8765`
3. 在多個瀏覽器分頁中開啟 `VCC_Clipper.html`

### 測試步驟

| 步驟 | 動作 | 預期結果 |
|------|------|----------|
| **1. 配對連線** | 分頁 A 點「建立配對」→ 分頁 B 輸入代碼點「加入配對」 | 兩端狀態顯示「🟢 已連線」 |
| **1b. 多人配對** | 分頁 C、D 輸入相同代碼點「加入配對」 | 四人各自顯示「🟢 已連線」，顯示名稱自動分配 |
| **2. 聊天室** | 分頁 A 輸入文字後按 Enter | 分頁 B、C、D 同時收到訊息，顯示發送者名稱與時間 |
| **2b. 顯示名稱** | 點擊頂部的顯示名稱 → 編輯新名稱 → 按 Enter | 下一則訊息以新名稱顯示 |
| **3. 檔案傳送** | 分頁 A 拖放一個檔案到傳輸區 | 分頁 B 自動下載該檔案 |
| **3b. 指定對象傳檔** | 分頁 A 選取特定對象再拖放檔案 | 僅該對象收到檔案 |
| **3c. 多人同時傳檔** | 分頁 A 傳給 B，分頁 C 同時傳給 D | 兩組傳輸互不干擾 |
| **4. 中斷連線** | 任一端點「中斷」 | 該端顯示「🔴 未連線」，其他端 peer 數減少 |
| **5. 重新配對** | 重新加入相同房間 | 所有功能正常運作 |
| **6. 伺服器中斷** | 連線狀態下 Ctrl+C 關閉信令伺服器 | 顯示「⚠ 信令伺服器離線」提示 |

### 預期行為

- 配對失敗時顯示明確的錯誤訊息（房間已滿、代碼錯誤等）
- 連線中斷時自動清理狀態，可重新配對
- 所有 DataChannel 錯誤顯示在 UI 中（非僅主控台）
- 檔案傳輸失敗顯示錯誤狀態與重試按鈕

## 專案架構

```
├── VCC_Clipper.html      # 主用戶端 SPA（WebRTC Full Mesh + 所有功能模組）
├── signal_server.py      # 信令伺服器（WebSocket N 人群組配對）
├── README.md             # 本文件
```

### 通訊協議

- **信令階段**：客戶端 ↔ WebSocket ↔ 信令伺服器（配對碼 + ICE 交換，不過資料）
- **資料階段**：客戶端 ↔ WebRTC DataChannel ↔ 客戶端（Full Mesh P2P 直連，DTLS 加密）

## 已知限制

- 多人同時傳檔時，每個對象同時只能傳一個檔案
- 需要 Python 3.8+ 執行信令伺服器
- 跨子網需要網路設備支援 UDP 路由（STUN/TURN 未實作）
- 房間上限 50 人（信令伺服器限制，可調整）
- 多人模式下成員離線須重新加入房間
